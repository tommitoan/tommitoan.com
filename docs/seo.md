# SEO Reference

## How metadata works

All route metadata is generated by `src/lib/metadata.ts:createMetadata()`.
It produces title, description, canonical URL, OpenGraph, and Twitter Card tags from a single call.

Root-level defaults (title template, global keywords, JSON-LD) live in `src/app/layout.tsx`.
Per-route overrides are set in each `page.tsx` via `export const metadata = createMetadata({...})`.

### Options

| Field | Type | Default | Notes |
|-------|------|---------|-------|
| `title` | `string` | site title | Used in `<title>` and OG title |
| `description` | `string` | site description | Used in meta description and OG |
| `path` | `string` | `"/"` | Sets canonical URL and OG URL |
| `keywords` | `string[]` | — | Page-specific keyword list |
| `robots` | `{ index, follow }` | — | Omit to inherit root default (index/follow) |
| `image` | `{ url, width, height, alt }` | `/social-card.png` (1200×630) | Per-route OG + Twitter image override |

### Example — adding a custom OG image to a route

```ts
export const metadata = createMetadata({
  title: "My Page",
  path: "/my-page/",
  image: {
    url: "/social-card-my-page.png",
    width: 1200,
    height: 630,
    alt: "My Page — Tommi Toan",
  },
});
```

## Sitemap rules

`src/app/sitemap.ts` generates `sitemap.xml` at build time.

**Only include indexable routes.** If a route has `robots: { index: false }` in its metadata, remove it from the sitemap.

Current sitemap entries:

| URL | Priority | Frequency | Last modified |
|-----|----------|-----------|---------------|
| `/` | 1.0 | weekly | 2025-06-01 |
| `/tech/` | 0.9 | monthly | 2025-06-01 |

`/discover/` and `/fengshui/` are excluded until their content is ready.

### When to add a route back

1. Content is live and meaningful (not a placeholder)
2. Remove `robots: { index: false }` from the page's `createMetadata()` call
3. Add the URL entry to `sitemap.ts` with an appropriate priority and a real date

```
User-Agent: *
Allow: /
Sitemap: https://tommitoan.com/sitemap.xml
```

No `Disallow` rules. All routes are crawlable. Indexing is controlled per-page via `robots` metadata.

## JSON-LD structured data

Two global schemas are injected in `src/app/layout.tsx`:

- `Person` — name, job title, location, social profiles, knowsAbout
- `WebSite` — site name and URL

The `/tech` page additionally injects a `ProfilePage` schema in `src/app/(content)/tech/page.tsx`.

To add structured data to a new route, add a `<script type="application/ld+json">` block directly in the page component (see `/tech` page for the pattern).

## Skill icons

All skill icons are stored locally in `public/skills/` as SVG files.
Paths are defined in `src/content/portfolio.ts` under the `skills` array.

Do not use CDN URLs (jsDelivr, simpleicons, etc.) — download the SVG to `public/skills/` first.

Current icons: `go`, `typescript`, `javascript`, `grpc`, `kafka`, `rabbitmq`, `postgresql`, `redis`, `mongodb`, `arangodb`, `aws`, `gcp`, `kubernetes`, `docker`, `jenkins`, `argocd`, `github-actions`, `helm`, `prometheus`, `github-copilot`, `claude`

## Analytics

Cloudflare Web Analytics is the recommended solution — no code changes required.

Enable it at: Cloudflare Dashboard → your domain → Web Analytics → Enable.

It tracks page views, unique visitors, top pages, and referrers. Privacy-first, no cookies, no banner needed.
